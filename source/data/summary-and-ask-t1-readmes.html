    Demo summary and discussion topics – June 29, 2026 meeting

    Summary of the demo

    • Package pages and guides will remain where they are.
    • In the package pages, only links are updated.
    • Guides, starting in 2025, will have a table at the bottom of the page containing all formats from the last 10 years.
      • Currently, old guides will not be updated.
      • The table in each new guide will get updated each year.
      • 5013-G is the exception. It will have one readme for all years, like the forms.
    • A single form readme will replace up to 10 from the last 10 years.
    • When the user comes to the forms readme from a prior-year package:
      • They will be taken directly to the table with the current year filtered to their year, using a URL parameter such as ?yr=2018&rgn=mb#prvs-yrs.
      • They will also see a paragraph below the table that can rescue them back to the package they came from.
    • The new forms will be listed in the forms listing page.
      • They will be architected under CRA forms listed by number.
      • The breadcrumbs will be updated.
      • Every readme will have a “You may be looking for” rescue link under the H1: All personal income tax packages.

    What we ask from you

    • Provide feedback on proposed changes. Keep in mind that things are very interconnected in these solutions.
    • Provide approval to change the H1s of the forms listing page as follows, for cleaner and more consistent H1s and breadcrumbs:
      • Canada Revenue Agency - CRA forms listed by number
      • Formulaires de l’Agence du revenu du Canada - ARC classés par numéro de formulaire
    • Provide new titles and metadata to be entered into the EDDS database:
      • H1 and title: they will be the same. Use the suggested endings below.
      • Intro or description: ...
      • Keywords: optional. Words that do not make it to the H1 can be added as keywords instead.
      • Subject: ...
    • Provide approval for current existing prior-year links to forms to be redirected to the unfiltered table in the corresponding new readmes directly. Current-year links can go to the top of the readme.
    • Provide approval for link changes in prior-year guides:
      • Each guide has a “Previous-year versions” or “Previous-year information” link at the bottom that takes the user back to “All personal income tax packages,” causing circular navigation.
      • I recommend these links be changed to the “Get another format or a previous-year version” section at the bottom of the 2025 guide, which contains a complete table for every prior year.
      • These should not need to be updated again until they are phased out. The destination tables will be updated each year. Not mocked up.

    Suggested H1 endings for T1 federal forms

    The following suggestions are made considering clarity, consistency and brevity, to be discussed. I believe provincial forms are clear enough and do not need these details, but correct me if they are not.

    When it is for all regions or groups, for example 5000-S7

    Suggesting we should not specify provinces and territories, because then we have to say non-residents too. It also does not need to be that clear because the tax package pages specify which forms are needed for each year.

    • (for all)
    • (pour tous)

    When it is for all, but with exceptions, for example 5000-S2

    • (for all except QC and non-residents)
    • (pour tous sauf QC et non-résidents)

    When provinces are specified and vary by year, for example 5000-R

    • (for multiple provinces and territories)
    • (pour plusieurs provinces et territoires)

    For one province only, for example 5009-R

    • (for AB)
    • (pour AB)

    Exceptions

    The uniqueness of the H1 is more important, so some exceptions might be warranted, for example 5000-R and 5015-R, to be discussed.
